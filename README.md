# ParkRiyadh — Vue 3 (Frontend only, no backend)

تطبيق حجز مواقف سيارات في الرياض، محوّل بالكامل من تصميم Figma (React) إلى **Vue 3 + Vite**، بدون أي باك إند — كل البيانات وهمية (mock) ومخزّنة محلياً في `src/data.js`.

## تشغيل المشروع

```bash
npm install
npm run dev
```

بعدها افتح الرابط اللي يطلع لك بالتيرمنال (عادة `http://localhost:5173`).

## البناء للإنتاج

```bash
npm run build
npm run preview
```

## المميزات
- 🌗 **دارك مود / لايت مود**: زر الشمس/القمر أعلى يمين الشريط العلوي (الحالة الافتراضية: دارك). خرائط الموقع نفسها تتغير بين تصميم داكن/فاتح تبع طبقات الخرائط.
- **خريطة حقيقية لمدينة الرياض** (Leaflet + OpenStreetMap/CARTO)، فيها زوم وسحب فعلي، وماركرات حية تتحرك حسب التوفر.
- تكبير/تصغير وزر "رجّعني لمركز الرياض" شغالين فعلياً على الخريطة.
- إذا ضغطت على أي مكان بالخريطة بعيد عن مواقعنا الحالية، يطلع بوب أب يقول إن هالموقع لسه ما شملته الخدمة وتحت التطوير.
- تدفق حجز كامل: التاريخ والوقت → المركبة → الدفع → التأكيد → QR Code.
- صفحات: الخريطة، تفاصيل الموقف، حجوزاتي، لوحة المستخدم، لوحة تحكم الأدمن (رسوم بيانية).
- كل شي responsive وواجهة عربية/إنجليزية جاهزة للتخصيص لاحقاً.

## ملاحظات تقنية
- Vue 3 + Vite + Tailwind CSS v4 (نفس نظام الألوان والـ tokens من تصميم الفيقما الأصلي).
- الأيقونات: `lucide-vue-next`.
- الرسوم البيانية (Revenue chart, Donut chart) مبنية بـ SVG مخصص بدل recharts (ما له نسخة Vue رسمية) — نفس الشكل تقريباً بدون اعتمادية إضافية.
- الأنيميشن مبسّط بـ CSS transitions بدل framer-motion (كانت React-only).
- ما فيه باك إند ولا قاعدة بيانات — كل البيانات mock في `src/data.js`، لما نضيف الباك إند الحقيقي (Spring Boot) بنربطها بـ API calls بدل البيانات الثابتة.
